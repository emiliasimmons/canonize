name: stash
description: Park a thought or to-do into the inbox for later, without acting on it now. Use only when the user explicitly asks to stash, park, or set aside an idea, or when grill-with-docs defers a tangent. Do NOT use when "stash" or "track" appear in passing, or when the user wants the idea acted on now rather than parked.
---

Park a thought so it is not lost, with no ceremony and no decision about what it is. Write it to the inbox as a timestamped scrap. That is all; do not act on it now.

This is the place for a thought that arrives mid-grill that you do not want to chase, or one that arrives from outside a session.

The inbox is plain scratch. To clear it, `maintain-docs` triages it, routing each scrap to where it belongs.

If an external capture channel is wired, this skill owns it. Otherwise the inbox is just a folder you write to.
